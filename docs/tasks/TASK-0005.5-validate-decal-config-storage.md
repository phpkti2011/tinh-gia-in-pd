## 4. validateDecalConfig được wire vào hàm nào?

| Hàm | Vị trí | Loại | Hành vi khi invalid |
|---|---|---|---|
| `saveDecalConfig(config)` | sync, gate trước localStorage.setItem | Sync save | return `false` + `console.warn` |
| `saveConfigToCloud('decalConfig', config, pw)` | gate trước cả localStorage + cloud | Async save | resolve `{local:false, cloud:false, error: '...'}` |
| `loadDecalConfig()` | deep check sau shallow `isValidConfig` | Sync load | fallback DECAL_DEFAULT_CONFIG + warn |
| `loadConfigFromCloud('decalConfig')` | deep check ở cả cloud branch + localStorage branch | Async load | fallback default + warn |

Module khác (`printConfig`, `largePrintConfig`, `uvdtfConfig`): **không thay đổi** — chỉ decal được wire.

## 5. Khi save invalid

| Lớp | Xử lý |
|---|---|
| `saveDecalConfig` | return `false`, không ghi localStorage |
| `saveConfigToCloud('decalConfig', …)` | return `{local: false, cloud: false, error: 'Decal config invalid: …'}` |
| `DecalSettingsPanel.handleSave` | hiển thị `alert('Cấu hình decal không hợp lệ, không lưu...')` rồi return — KHÔNG gọi `onSave` |
| App.jsx `onSave` của Decal | không bao giờ được gọi → `setConfig` + `saveConfigToCloud` không chạy → React state giữ config cũ → UI nhất quán |

→ Không crash, không màn hình trắng. Admin biết ngay lỗi qua alert + console.

## 6. Khi load invalid

| Tình huống | Xử lý |
|---|---|
| `localStorage.decalConfig` là JSON corrupted | catch → `console.error` → fallback `DECAL_DEFAULT_CONFIG` |
| `localStorage.decalConfig` thiếu key (fail `isValidConfig`) | warn shallow → fallback default |
| `localStorage.decalConfig` pass shallow nhưng FAIL deep schema (TASK-0005.5) | warn deep → fallback default |
| Cloud trả về config invalid | warn → fallback localStorage → nếu localStorage cũng invalid → default |

**KHÔNG xoá localStorage** khi fallback (theo yêu cầu user). Dữ liệu xấu vẫn còn để dev debug.
